# MUNDUS LAB — User Model

Das User Model beschreibt, wie Nutzer mit MUNDUS interagieren, welche Rollen sie einnehmen, welche Handlungen sie ausführen und wie das System aus ihrem Verhalten lernt.  
Der Nutzer ist nicht Konsument, sondern aktiver Mitgestalter des Wissenssystems.

MUNDUS basiert auf dem Grundprinzip:  
**Nutzer verbessern das System, indem sie es verwenden.**

---

## 1. Grundprinzip des Nutzerverhaltens

Nutzer handeln aus **intrinsischer Motivation**.  
Sie verbessern Steine nicht für Belohnungen, sondern weil sie präzise Ergebnisse brauchen:

- um Patente zu verstehen  
- um Technik zu analysieren  
- um Lösungen zu konstruieren  
- um Alternativen zu finden  
- um Fehler zu vermeiden  

Je besser MUNDUS sie unterstützt, desto mehr unterstützen sie MUNDUS.

---

## 2. Nutzerrollen im System

Ein Nutzer kann mehrere Rollen gleichzeitig einnehmen:

### 2.1 Explorer  
Erkundet den Graphen, sucht, navigiert, lernt.

### 2.2 Analyst  
Untersucht Systeme, vergleicht Steine, bewertet technische Zusammenhänge.

### 2.3 Konstrukteur  
Kombiniert Steine, entwickelt Systeme, erschafft neue technologische Varianten.

### 2.4 Korrektor  
Verbessert Roh-Steine, korrigiert Beziehungen, ergänzt fehlende Informationen.

### 2.5 Erzeuger  
Legt neue Steine an, die bisher im System fehlen.

Dies bildet das Fundament für HRM und Graph-Wachstum.

---

## 3. Nutzeraktionen

Der Nutzer kann folgende Aktionen ausführen:

- Steine ansehen  
- Steine vergleichen  
- Steine kombinieren  
- Steine korrigieren  
- neue Steine erstellen  
- Beziehungen anpassen  
- Graph navigieren  
- Vorschläge annehmen  
- Vorschläge ablehnen  
- Systeme konstruieren  
- Systeme dekonstruieren  
- Varianten testen  
- Alternativen prüfen  

Jede dieser Aktionen erzeugt Feedback für HRM.

---

## 4. Wie der Nutzer das System verbessert

### 4.1 Durch Korrektur
Fehlerhafte Roh-Steine werden durch Nutzer:
- neu kategorisiert  
- präzisiert  
- strukturiert  
- mit besseren Beziehungen versehen  

### 4.2 Durch Konstruktion
Jede gültige Kombination zeigt:
- welche Steine kompatibel sind  
- welche Funktionsketten realistisch sind  
- welche Mechanismen üblich sind  

### 4.3 Durch Ablehnen
Abgelehnte Vorschläge verbessern:
- Sandbox-Logik  
- HRM-Gewichtungen  
- zukünftige Zerlegungen  

### 4.4 Durch Navigation
Häufige Navigationspfade zeigen technologische Muster.

### 4.5 Durch Stein-Erstellung
Neue Steine erweitern den Wissensgraph gezielt.

---

## 5. Psychologisches Modell

Der Nutzer:
- sucht Klarheit,  
- sucht Verständnis,  
- sucht Lösungen,  
- sucht Optimierungen,  
- sucht funktionierende Konstruktionen.  

Er handelt **zielgerichtet** und **wissensgetrieben**.  
Das System muss ihn dabei **unterstützen**, niemals behindern.

---

## 6. Systemanforderungen aus Nutzersicht

Das System muss:

- verständlich sein  
- logisch strukturiert sein  
- fehlerkorrigierbar sein  
- schnelle Rückmeldung geben  
- visuell klar sein  
- technische Zusammenhänge sichtbar machen  
- Alternativen klar darstellen  

Nichts davon darf kompliziert oder überladen wirken.

---

## 7. Nutzergrenzen (was der Nutzer nicht tun muss)

Nutzer müssen NICHT:

- Steinformat verstehen  
- Algorithmen kennen  
- KI-Modelle trainieren  
- Code schreiben  
- Graphregeln lernen  

Das System übernimmt die Komplexität.  
Der Nutzer arbeitet rein logisch.

---

## 8. Ziel des User Models

Das User Model hat ein einziges Ziel:

**Der Nutzer arbeitet so, wie ein Mensch arbeitet —  
und das System lernt so, wie eine Maschine lernen muss.**

Es bildet die Brücke zwischen menschlicher Intuition und maschineller Struktur.
