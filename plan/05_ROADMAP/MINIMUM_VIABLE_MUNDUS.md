# MUNDUS LAB — Minimum Viable Mundus (MVM)

Dieses Dokument definiert die kleinste mögliche Version von MUNDUS,  
die bereits vollständig funktionsfähig ist und den Kern des Systems bildet.  
Alle späteren Erweiterungen bauen auf diesem Minimalzustand auf.

Das Minimum Viable Mundus (MVM) ist der erste Moment,  
in dem „MUNDUS lebt“ und Nutzer echten Nutzen daraus ziehen können.

---

## 1. Ziel des MVM

Ein funktionsfähiges System bereitzustellen, das:

- Patente und technische Texte aufnehmen kann  
- sie in Roh-Steine zerlegt  
- Nutzern ermöglicht, Steine zu korrigieren  
- Steine in den Graph integriert  
- Werkstatt bietet, um Steine zu analysieren  
- einfache Sandbox-Vorschläge liefert  
- HRM aus Korrekturen lernt  

Dieses Minimum macht MUNDUS sofort nutzbar,  
ohne dass das volle gesamte System gebaut sein muss.

---

## 2. Komponenten, die im MVM enthalten sein müssen

### 2.1 Roh-Stein-Zerlegung (Basic Decomposition)
- Extraktion von Aussagen  
- Vorläufige Typzuweisung  
- Erstellung von Roh-Steinen  
- Patent → Steinliste  

### 2.2 Stein-Editor (Validation Interface)
Der Nutzer muss:
- Typ korrigieren  
- summary anpassen  
- Inhalte verbessern  
- Beziehungen hinzufügen oder löschen  

### 2.3 Validierungslogik
- Format prüfen  
- Pflichtfelder prüfen  
- Beziehungen prüfen  
- Versionierung setzen (1.0.0)  

### 2.4 Wissensgraph (Simple Graph Engine)
Der Graph muss:
- Knoten speichern  
- Kanten speichern  
- einfache Navigation erlauben  

### 2.5 Werkstatt (Basic Workshop)
Muss folgende Grundfunktionen haben:

- Einzelsteinansicht  
- Korrekturmodus  
- einfache Kombination (Drag-and-Drop)  
- Anzeige von Beziehungen  
- visueller Graphbrowser  

### 2.6 Sandbox (Minimum Logic)
Die Sandbox muss im Minimum:

- Kompatibilität prüfen  
- einfache Konflikte erkennen  
- alternative Steine vorschlagen (auf Basis von Kategorie)  
- fehlende Inputs melden  

Mehr nicht.

### 2.7 HRM (Basic Learning)
HRM muss im Minimum:

- Korrekturen registrieren  
- Annahme/Ablehnung von Vorschlägen registrieren  
- Häufigkeit von Kombinationen speichern  
- einfache Priorisierungen lernen  

---

## 3. Was das MVM bewusst NICHT enthält

- keine komplexen Optimierungsalgorithmen  
- keine erweiterten Variantenanalysen  
- keine erweiterten Visualisierungen  
- keine Material-/Energie-/Kostenmodelle  
- keine Teamfunktionen  
- keine Community-Features  
- keine Simulationen  
- keine API für externe Systeme  
- keine automatischen Steinverbesserungen  
- keine vollautomatische Zerlegung  

Das MVM ist radikal minimal, um schnell nutzbare Ergebnisse zu liefern.

---

## 4. Minimaler Nutzerfluss (MVM User Flow)

1. Nutzer lädt Patent hoch  
2. Roh-Steine werden erzeugt  
3. Nutzer korrigiert Steine  
4. Validierte Steine kommen in den Graph  
5. Nutzer öffnet Werkstatt  
6. Nutzer kombiniert Steine  
7. Sandbox prüft  
8. HRM lernt aus Aktionen  
9. MUNDUS wird besser  

Das ist das technisch kleinste funktionierende Ökosystem.

---

## 5. Was Nutzer im MVM bereits tun können

- Patente aufschlüsseln  
- Mechanismen verstehen  
- Funktionsketten erkennen  
- Steine verbessern  
- einfache technische Systeme bauen  
- Alternativen finden  
- Fehler erkennen  
- Wissen strukturiert erfassen  

Schon das ist ein mächtiges Werkzeug.

---

## 6. Qualitätskriterien für das MVM

Das MVM ist **erfolgreich**, wenn:

- Nutzer Steine korrekt validieren können  
- der Graph konsistent bleibt  
- die Werkstatt stabil läuft  
- die Sandbox sinnvolle Vorschläge erzeugt  
- HRM Verbesserungen zeigt  
- Nutzer echte technische Klarheit erhalten  

---

## 7. Zusammenfassung

Das Minimum Viable Mundus besteht aus:

1. Roh-Zerlegung  
2. Stein-Validierung  
3. Wissensgraph  
4. Werkstatt (Basis)  
5. einfache Sandbox  
6. einfaches HRM  
7. minimaler Nutzerfluss  

Es ist die kleinste funktionale Einheit eines globalen technischen Wissenssystems.

Von hier aus wächst MUNDUS unbegrenzt weiter.
