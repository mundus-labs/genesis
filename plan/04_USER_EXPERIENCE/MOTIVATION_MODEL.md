# MUNDUS LAB — Motivation Model

Dieses Dokument beschreibt das Motivationsmodell von MUNDUS.  
Es erklärt, warum Menschen das System gerne nutzen,  
warum sie freiwillig qualitativ hochwertige Steine erzeugen  
und warum das System ohne externe Belohnungen (wie Tokens, Punkte oder Gamification) funktioniert.

Der Schlüssel:  
**MUNDUS gibt dem Nutzer direkt einen hohen Nutzen zurück.**

---

## 1. Grundidee des Motivation Models

Menschen arbeiten an MUNDUS mit, weil sie:

1. etwas verstehen wollen,  
2. etwas konstruieren wollen,  
3. etwas lösen wollen,  
4. etwas verbessern wollen,  
5. etwas entdecken wollen,  
6. echte technische Klarheit suchen.  

Das System motiviert also durch ZWECK, nicht durch Gamification.

---

## 2. Die drei Motivationsquellen

### 2.1 Intrinsische Motivation
Nutzer arbeiten an MUNDUS, weil:
- sie technische Klarheit wollen  
- sie ein Patent verstehen müssen  
- sie eigene Systeme bauen wollen  
- sie Alternativen testen möchten  
- sie bessere Resultate brauchen  

Das System verstärkt diese Motivation durch:
- klare Darstellung  
- direkte Verbesserungen  
- sinnvolle Werkzeuge  

### 2.2 Kompetenzgewinn
Jeder Nutzer spürt:
- ich verstehe Technik besser  
- ich kann Systeme logisch zerlegen  
- ich kann Mechanismen vergleichen  
- ich sehe Zusammenhänge, die sonst unsichtbar bleiben  

Das motiviert, weil es echte persönliche Fortschritte liefert.

### 2.3 Systemischer Nutzen
Jeder Beitrag verbessert:
- Sandbox-Vorschläge  
- HRM-Lernen  
- Steinqualität  
- Graphstruktur  

Und Nutzer merken:
**Je mehr ich beitrage, desto besser funktioniert MUNDUS für mich selbst.**

---

## 3. Warum keine Tokens oder Belohnungssysteme?

Ein technisches Wissenssystem lebt von:
- Klarheit  
- Logik  
- Präzision  
- Vertrauen  

Tokens würden:
- Motivation verfälschen  
- Qualität zerstören  
- Spam erzeugen  
- Nutzer in falsche Richtung drängen  
- Fokus von Technik zu „Punkten“ verschieben  

MUNDUS basiert stattdessen auf **Wert durch Nutzen**.

---

## 4. Psychologische Motivationselemente

### 4.1 Klarheit (Transparenz)
Menschen lieben Systeme, die verständlich sind.

### 4.2 Kontrolle (Agency)
Werkstatt gibt volle Kontrolle → Nutzer entscheidet, System folgt.

### 4.3 Fortschritt (Progress)
Jede Korrektur verbessert das System sichtbar.

### 4.4 Kompetenz (Mastery)
Nutzer bauen Wissen auf und erleben Erfolgserlebnisse.

### 4.5 Neugier (Exploration)
Der Graph lädt zum Entdecken ein — neue Zusammenhänge wirken belohnend.

---

## 5. Warum Nutzer freiwillig Steine korrigieren

Sie korrigieren Steine, weil:

1. sie selbst bessere Ergebnisse wollen  
2. unklare Steine sie beim Arbeiten stören  
3. korrigierte Steine sofort bessere Vorschläge liefern  
4. die Werkstatt durch präzise Steine leichter wird  
5. das HRM zukünftige Fehler vermeidet  

Ein korrigierter Stein ist **sofort** ein persönlicher Vorteil.

---

## 6. Warum Nutzer die beste Datenqualität liefern

Der Nutzer ist der Experte für:
- den Kontext seines Problems  
- die richtige Bedeutung eines Steins  
- die korrekte mechanische oder funktionale Zuordnung  

MUNDUS nutzt diese menschliche Expertise optimal, indem es:
- Fehler sofort sichtbar macht  
- Korrekturen einfach ermöglicht  
- Nutzen direkt zurückgibt  

---

## 7. Systemverhalten, das Motivation unterstützt

### 7.1 Sofortige Rückmeldung
Jede Korrektur verbessert:
- Sandbox-Vorschläge  
- Graphqualität  
- HRM-Wissen  
innerhalb weniger Sekunden.

### 7.2 Sichtbarer Fortschritt
Graph wird größer, präziser, strukturierter.

### 7.3 Keine Strafen
Ablehnung eines Vorschlags ist kein Fehler, sondern ein Lernsignal.

### 7.4 Keine Überforderung
Komplexität wird gebrochen in kleine Steine und klare UI.

---

## 8. Langzeitmotivation

MUNDUS motiviert langfristig, weil:
- der Wissensgraph wächst  
- die Sandbox besser wird  
- der Nutzer immer schneller zu Lösungen kommt  
- komplexe Technik immer verständlicher wird  

Menschen bleiben, weil das System **echten Wert erzeugt**.

---

## 9. Zusammenfassung

Das Motivation Model basiert auf:

- intrinsischer Motivation  
- Kompetenzgewinn  
- sofortigem Nutzen  
- klarer UI  
- Transparenz  
- menschlicher Kontrolle  
- systemischem Fortschritt  

MUNDUS braucht keine künstlichen Anreize.  
Der Anreiz ist die Werkstatt selbst — die perfekte Maschine zur technischen Klarheit.
